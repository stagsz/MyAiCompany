## Goal

Build AuditFlow V2 on top of the complete V1 foundation (108/108 tasks, 151 tests). V2 makes the AI the engine: five Hermes/Claude loops run continuously to score clause compliance, close CAPAs, classify evidence, benchmark peers, and grow an organizational knowledge brain. V1 REST API and frontend are unchanged; V2 adds a new `/api/ai/*` surface, Prisma schema additions, Hermes agent YAML configs, and new frontend panels.

---

## Current Context

**Stack:** Next.js 14 (App Router), Express/TypeScript, Prisma 5, Supabase (PostgreSQL 15 + RLS + Storage), JWT auth, RBAC.

**What already exists in `/backend/src`:**
- `services/`: assessmentService, evidenceService, nonConformityService, correctiveActionService, reportService, etc.
- `controllers/` + `routes/`: full REST coverage for assessments, NCRs, CAPAs, evidence, users, templates.
- `index.ts`: already exports `app` as default and conditionally calls `listen()` — Vercel-compatible.
- Prisma schema: `Assessment`, `QuestionResponse`, `Evidence`, `NonConformity`, `CorrectiveAction`, `User`.

**What is missing (all V2):**
- Anthropic SDK not in `package.json`
- No `Organization` multi-tenant model in Prisma
- No `ClauseScore`, `AiLoopRun`, `EvidenceDocument`, `KnowledgeBrain` tables
- No pgvector on Supabase
- No `/api/ai/*` routes
- No Hermes loop YAML files
- No Vercel cron endpoints
- No benchmark aggregation tables

**Deployment blocker:** `DEPLOY-001` — Vercel vercel.json routes need fixing. `index.ts` already exports correctly; `vercel.json` needs the routes + crons config.

---

## Proposed Approach

Execute the PRD phases in order: Foundation → Assessment Loop → CAPA Loop → Evidence + Knowledge Loops → Benchmark + SaaS Layer → Deployment. Each phase is independently deployable. Use Claude (`claude-sonnet-4-20250514`) via Anthropic SDK for all AI work. Hermes cron jobs handle loop scheduling. Supabase Realtime webhooks trigger per-INSERT loops.

## Step-by-Step Plan

### Phase 1 — AI Foundation (Tasks AI-001 to AI-006)

1. **AI-001** — Install Anthropic SDK  
   - `npm install @anthropic-ai/sdk` in `backend/`  
   - Create `backend/src/services/ai.service.ts` — base Claude client wrapper  
     - Reads `ANTHROPIC_API_KEY` from env  
     - Exposes `callClaude(systemPrompt, userPrompt, maxTokens)` → returns text  
     - Increments `tokensUsedThisMonth` on `Organization` after each call  

2. **AI-002** — Prisma migrations: new V2 models  
   - Add to `prisma/schema.prisma`:
     ```
     model Organization { id, name, industrySegment, subscriptionTier, tokenBudgetMonthly, tokensUsedThisMonth, benchmarkOptIn, createdAt + relations }
     model ClauseScore { id, assessmentId, organizationId, clauseNumber, maturityScore 1-5, confidence, gapSummary, evidenceCoverage, recommendedActions Json, scoredAt }
     model AiLoopRun { id, loopName, organizationId, triggerType, status, tokensUsed, resultSummary Json, escalatedToHuman, durationMs, createdAt }
     model EvidenceDocument { id, organizationId, storagePath, documentType, clausesCovered String[], relevanceScore, aiClassification Json, requiresReview, createdAt }
     ```
   - Run `npx prisma migrate dev --name v2-ai-foundation`  

3. **AI-003** — Enable pgvector on Supabase + KnowledgeBrain table  
   - SQL in Supabase editor:
     ```sql
     CREATE EXTENSION IF NOT EXISTS vector;
     CREATE TABLE knowledge_brain (id UUID PK, organization_id UUID FK, section TEXT, content_md TEXT, embedding vector(1536), source_type TEXT, last_updated TIMESTAMPTZ DEFAULT now());
     ALTER TABLE knowledge_brain ENABLE ROW LEVEL SECURITY;
     CREATE POLICY "org_isolation" ON knowledge_brain FOR ALL USING (organization_id = auth.uid());
     CREATE INDEX ON knowledge_brain USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
     ```
   - Note: `knowledge_brain` lives outside Prisma (pgvector not yet fully supported). Use raw Supabase client for vector ops.  

4. **AI-004** — RLS on all new AI tables  
   - Add `ENABLE ROW LEVEL SECURITY` + `organization_id = auth.uid()` policy to: `clause_scores`, `ai_loop_runs`, `evidence_documents`  
   - Add these as SQL migrations (Supabase SQL editor or Prisma `db push` + manual RLS)  

5. **AI-005** — `/api/ai/*` router + token budget middleware  
   - Create `backend/src/routes/aiRoutes.ts` — register under `/api/ai`  
   - Create `backend/src/middleware/tokenBudget.ts`:
     - Look up org's `tokenBudgetMonthly` vs `tokensUsedThisMonth`
     - Return 429 if over budget
     - Pass org to req context  
   - Register `aiRoutes` in `backend/src/routes/index.ts`  

6. **AI-006** — Supabase Realtime webhook to loop trigger  
   - In Supabase Dashboard → Database → Webhooks:  
     - Table: `assessment_responses`, Event: `INSERT` → POST to `/api/loops/trigger/assessment`  
     - Table: `non_conformities`, Event: `INSERT` → POST to `/api/loops/trigger/capa`  
   - Create `backend/src/routes/loopRoutes.ts` with `/trigger/*` and `/cron/*` endpoints  
   - Register under `/api/loops`

### Phase 2 — Assessment Loop (Tasks ASMT-001 to ASMT-007)

7. **ASMT-001** — ISO 9001:2015 question bank JSON  
   - File: `backend/src/data/questionBank.json`  
   - Structure: `{ "4.1": { title, questions: [{id, text, evidenceRequired, maturityDescriptors: {1:..,5:..}}], primaryLoop } }`  
   - Cover all 24 clauses from PRD clause map (~180 questions total)  

8. **ASMT-002** — `analyzeClause()` service function  
   - `backend/src/services/assessmentAiService.ts`  
   - Takes: `assessmentId`, `clauseNumber`, `responses[]`, `orgId`  
   - Builds system prompt from clause maturity descriptors  
   - Calls Claude → returns `{ maturityScore, confidence, gapSummary, evidenceCoverage, recommendedActions[] }`  
   - Writes to `clause_scores` table  
   - Logs run to `ai_loop_runs`  

9. **ASMT-003** — `POST /api/ai/assessment/:id/analyze` controller + `GET /api/ai/assessment/:id/scores`  

10. **ASMT-004** — Gap report generation  
    - Claude produces structured Markdown → convert to PDF via pdfkit (already in deps)  
    - Store in Supabase Storage bucket `reports/{orgId}/`  
    - Return signed URL  
    - Endpoint: `POST /api/ai/assessment/:id/report`  

11. **ASMT-005** — Frontend: AI Analysis Panel  
    - File: `frontend/src/app/(dashboard)/assessments/[id]/ai-panel.tsx`  
    - Shows maturity score per clause (1–5 badge), gap summary, recommended evidence, confidence bar  
    - Fetches from `GET /api/ai/assessment/:id/scores`  

12. **ASMT-006** — Frontend: Clause Maturity Radar Chart  
    - File: `frontend/src/components/charts/MaturityRadar.tsx`  
    - Recharts RadarChart — all clauses 4–10 on 1–5 scale  
    - Optional benchmark overlay (P50 line) from benchmark API  

13. **ASMT-007** — Hermes loop YAML + Vercel cron endpoint  
    - File: `agents/assessment-loop.yaml` (YAML config as per PRD §07)  
    - Endpoint: `POST /api/loops/cron/assessment` — runs weekly gap scan across all orgs  
    - Add to `vercel.json` crons: `{ "path": "/api/loops/cron/assessment", "schedule": "0 6 * * 1" }`

### Phase 3 — CAPA Loop (Tasks CAPA-001 to CAPA-005)

17. **CAPA-004** — Weekly pattern detection

## Risks, Tradeoffs, Open Questions

**Risk: Supabase pgvector** — pgvector must be enabled on the production Supabase instance. If on a plan that doesn't support it, knowledge_brain embeddings need a workaround (store as text, skip vector similarity for now).

**Risk: Token costs** — Running 5 loops per organization per week at scale will accumulate real API costs. The token budget middleware (AI-005) is critical. Start with conservative limits per tier.

**Risk: Supabase Realtime webhooks on hosted plans** — Realtime webhook delivery to Vercel serverless endpoints has cold-start latency. For MVP, the cron-based triggers are more reliable. Consider making Realtime webhooks a Professional-tier feature and using polling fallback for Starter.

**Tradeoff: pgvector vs plain text** — Full vector similarity requires text-embedding-ada-002 or similar. For MVP, store knowledge_brain as plain text (content_md) and use Claude directly for synthesis. Add vector indexing as a Phase 5+ enhancement once the feature value is proven.

**Open question: Organization model migration** — V1 appears single-tenant (no Organization FK on User/Assessment). The multi-tenant migration requires adding `organizationId` to existing tables. Needs careful migration to avoid breaking V1 data. Consider defaulting all existing records to a "default" org.

**Open question: Benchmark minimum sample size** — PRD specifies minimum 10 orgs per industry segment before publishing P25/P50/P75. Until that threshold is hit, show benchmark as "coming soon" in UI. Do not expose individual scores.

**Open question: Claude model pinning** — PRD specifies `claude-sonnet-4-20250514`. Confirm this is available on your Anthropic account tier before Phase 2.
