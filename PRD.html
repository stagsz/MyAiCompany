01 · VISION

Not a checklist.
A self-improving QMS.

AuditFlow v2 turns ISO 9001:2015 compliance from a periodic, human-driven audit exercise into a continuously self-improving AI system. Five recursive Hermes loops run 24/7 — evaluating evidence, closing gaps, recommending actions, and distilling organizational knowledge — so the quality posture gets stronger while everyone sleeps.

WHAT CHANGED IN V2

AI-Native, Not AI-Added

V1 was a well-built audit tool. V2 makes the AI the engine, not a feature. Every workflow runs through a Hermes agent loop with sensors, policies, tools, and a learning mechanism.

THE LOOP/OS PRINCIPLE

Burn Tokens, Not Headcount

Based on Tom Blomfield's YC framework: replace management hierarchy with recursive AI loops. For ISO compliance: replace the annual audit cycle with a continuous self-improving system.

02 · PROBLEM STATEMENT

ISO 9001 is a once-a-year event.
It should not be.

Every organization with ISO 9001 certification runs the same painful cycle: scramble for the internal audit, prepare for the external audit, close the NCRs found, and go back to sleep until next year. The standard requires continual improvement — the reality is episodic compliance theater.

PAIN 01

Evidence is scattered

Documents live in SharePoint, email threads, Slack, physical binders. Proving clause compliance requires manual archaeology every audit cycle.

PAIN 02

Gaps found too late

Compliance gaps surface during the external audit — not six months earlier when they could be fixed cheaply and without stress.

PAIN 03

CAPA never closes

Corrective actions get opened after every audit. A third are still open at the next audit. Root causes recur because no system learns from patterns.

PAIN 04

Knowledge walks out

The QMS lives in the heads of two people. When they leave, the institutional knowledge of how and why processes work disappears with them.

PAIN 05

Benchmarks are guesswork

Organizations have no way to know whether their maturity score is good or typical for their industry. Flying blind relative to peers.

PAIN 06

Management review: ceremony

Clause 9.3 review is a slide deck compiled the week before. Data is stale. Actions not tracked. The loop never actually closes.

AUDITFLOW V2 RESPONSE

Five autonomous AI loops replace the episodic audit cycle with continuous monitoring. Assessment Loop scores gaps weekly. CAPA Loop tracks actions and flags stalls. Evidence Loop tags and organizes documents in real time. Knowledge Loop builds the institutional brain. Benchmark Loop gives organizations a live view of how they compare.

03 · USER PERSONAS

Who uses this.

Four distinct user types, each with a different relationship to the AI loops.

PRIMARY USER
The Quality Manager
Responsible for ISO 9001 certification. Runs internal audits, manages NCRs, prepares management review. Wants to spend less time chasing evidence and more time actually improving processes.
Spends 60% of audit prep time collecting documents that already exist somewhere in the organization.
SECONDARY USER
The Internal Auditor
Conducts clause-by-clause assessments of departments. Needs structured question sets, evidence links, and a way to document findings without a 40-tab Excel nightmare.
No tool tells them what clause gaps look like in this specific organization's context.
TERTIARY USER
The Executive / DRI
Needs to understand compliance posture without reading audit reports. Approves CAPA budgets. Signs off on management review. Wants one-screen situational awareness.
Gets audit results two weeks after the audit. By then the window to act has passed.
PLATFORM USER
The SaaS Admin
Manages multiple organizations on the platform. Needs tenant isolation, consolidated reporting, benchmark visibility, and billing management across entities.
Today this means separate spreadsheets per entity with no cross-org insight or AI leverage.

Maturity Model (1–5)

LEVEL 1 — INITIAL

No documented process. Ad hoc, reactive. Relies on individual heroics.

LEVEL 2 — MANAGED

Process exists but not consistently followed. Partial documentation.

LEVEL 3 — DEFINED

Standardized, documented. Consistently followed. Evidence available.

LEVEL 4 — MEASURED

Monitored with KPIs. Trends tracked. Data-driven decisions made.

LEVEL 5 — OPTIMIZING

Continuous improvement built in. Proactive. Benchmarked against peers.

AI ROLE

Assessment Loop scores each clause, flags regression, drives maturity toward Level 5 via CAPA recommendations.

07 · HERMES AGENT CONFIGURATIONS

YAML configs
for each loop.

Each loop runs as an independent Hermes agent. Deploy under agents/ in the AuditFlow repo.

Assessment Loop

name: assessment-loop
model: claude-sonnet-4-20250514

persona: |
  ISO 9001:2015 Assessment Agent for AuditFlow.
  Analyze assessment responses, score each clause 1-5,
  identify gaps, recommend evidence to gather, generate
  gap analysis reports. Organizations get smarter
  assessments every cycle without additional human effort.

sensors:
  - id: assessment_responses
    type: webhook
    source: supabase_realtime
    table: assessment_responses
    event: INSERT
  - id: weekly_clause_review
    type: cron
    schedule: "0 6 * * 1"  # Monday 06:00 UTC

policy:
  maturity_scale: 1-5
  confidence_threshold: 0.8
  auto_generate_report: true
  require_human_review: [maturity_score_1, evidence_conflict]

tools:
  - supabase_db         # read/write assessment data
  - claude_api          # gap analysis, question generation
  - evidence_classifier # tag uploaded docs to clauses
  - report_generator    # PDF/HTML gap report output
  - notification_api    # alert QM on gaps found

quality_gate:
  response_confidence:
    threshold: 0.8
    fail_action: request_clarification
  human_review_maturity_1: { required: true }

learning:
  on_assessment_complete:
    - update_question_bank: true
    - contribute_to_benchmark: true
    - report_to_knowledge_loop: true
  monthly:
    - recalibrate_clause_weights: true

CAPA Loop

name: capa-loop
model: claude-sonnet-4-20250514

persona: |
  CAPA Intelligence Agent for AuditFlow.
  Monitor all open NCRs and corrective actions,
  identify recurring root causes, recommend SMART
  actions with owners and deadlines, close the loop
  on every finding. Prevent the same nonconformity
  from appearing twice.

sensors:
  - id: new_ncr
    type: webhook
    source: supabase_realtime
    table: non_conformities
    event: INSERT
  - id: capa_overdue
    type: cron
    schedule: "0 8 * * *"  # daily 08:00 UTC
    filter: "due_date < now() + interval '7 days' AND status != closed"

policy:
  severity_matrix:
    critical: { deadline_days: 14, escalate_immediately: true }
    major:    { deadline_days: 30, escalate_at: "80% of deadline" }
    minor:    { deadline_days: 60, escalate_at: "80% of deadline" }
  require_human_review:
    - systemic_ncr
    - customer_complaint_linked
    - capital_investment_required

tools:
  - supabase_db
  - claude_api          # 5-Why, Fishbone, SMART action generation
  - notification_api
  - calendar_api        # set deadline reminders for CAPA owners

learning:
  on_capa_closed:
    - archive_effective_action: true
    - update_root_cause_index: true
  on_recurring_pattern:
    - upgrade_to_proactive_suggestion: true
    - flag_to_knowledge_loop: true

Evidence Loop

name: evidence-loop
model: claude-sonnet-4-20250514

persona: |
  Evidence Intelligence Agent for AuditFlow.
  Every document uploaded gets classified, tagged to
  the relevant ISO clauses, and linked to the active
  assessment. If a clause has insufficient evidence,
  flag it before the auditor asks.

sensors:
  - id: document_upload
    type: webhook
    source: supabase_storage
    event: object.created
    bucket: evidence
  - id: coverage_gap_alert
    type: queue
    source: assessment-loop
    filter: type=evidence_gap

policy:
  relevance_threshold: 0.75
  auto_tag: true
  require_human_review:
    - relevance_score_below_threshold
    - sensitive_personal_data_detected
    - legal_contractual_document

tools:
  - supabase_storage    # read uploaded files
  - ocr_api             # extract text from PDFs and images
  - claude_api          # classify to ISO clause
  - supabase_db         # write tags and clause links

quality_gate:
  relevance_score: { threshold: 0.75, fail_action: flag_for_human }
  pii_check:       { required: true, fail_action: redact_and_notify }
  currency_check:  { max_age_years: 3, warn_if_exceeded: true }

name: knowledge-loop
model: claude-sonnet-4-20250514

persona: |
  Knowledge Distillation Agent for AuditFlow.

    event: UPDATE

policy:
  pii_scrubbing: strict
  retention_days: 730
  auto_ingest: true
  manual_regen_schedule: "0 3 * * 1"  # Monday 03:00 UTC

tools:
  - summarization_engine
  - vector_db           # pgvector on Supabase
  - skills_file_writer
  - management_review_generator
  - claude_api

learning:
  always: [update_company_brain]
  weekly: [distill_skills, regenerate_qms_manual, broadcast_context_refresh]
  on_threshold: [compress_context, prune_stale_entries]

DEPLOY-001 · ROOT CAUSE OF VERCEL FAILURE

Express on Vercel runs as a serverless function. The entry point must export the Express app object — not call app.listen(). Wrap with: if (process.env.NODE_ENV !== 'production') app.listen(PORT). Then vercel.json must point all /api/* routes to dist/index.js with outputDirectory: "dist". See Deployment section for the correct configuration.

Supabase SQL — Enable pgvector

-- Run in Supabase SQL Editor → New Query

-- Enable vector extension
CREATE EXTENSION IF NOT EXISTS vector;

-- Knowledge brain table with vector column
CREATE TABLE knowledge_brain (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id  UUID NOT NULL REFERENCES organizations(id),
  section          TEXT NOT NULL,
  content_md       TEXT NOT NULL,
  embedding        vector(1536),
  source_type      TEXT,
  last_updated     TIMESTAMPTZ DEFAULT now()
);

-- RLS: tenant isolation
ALTER TABLE knowledge_brain ENABLE ROW LEVEL SECURITY;
CREATE POLICY "org_isolation" ON knowledge_brain
  FOR ALL USING (organization_id = auth.uid());

-- Vector similarity index
CREATE INDEX ON knowledge_brain
  USING ivfflat (embedding vector_cosine_ops)
  WITH (lists = 100);
